# 用户体验分析

## 核心用户与需求
- 资产记录者：希望快速记录已购物品、维持账目清晰，并能够查看长期价值与使用效率。
- 计划购买者：在心愿单中跟踪价格走势与预算，减少冲动消费。
- 轻量管理者：希望操作简单、数据可控、离线可用。

## 关键痛点
- 录入成本高：传统手动录入步骤多、信息易缺失。
- 信息分散：分类、状态、渠道不统一导致统计不准确。
- 可视化弱：难以直观看到月度趋势与渠道占比。

## 核心体验目标
- 低摩擦录入：AI 快速添加 + 多图识别降低输入负担。
- 结构化管理：分类/状态/来源统一治理，保证全局一致。
- 可视化洞察：趋势、分布、TopN 一眼看懂。

## 关键用户路径
1. 已购录入：主界面进入添加 -> 填写必要字段 -> 保存。
2. AI 快速录入：上传图片/文字 -> AI 填充 -> 用户确认。
3. 心愿单管理：搜索 + 筛选 -> 价格记录 -> 观察走势。
4. 设置与维护：AI 设置 -> 数据管理 -> 统一维护分类/状态/来源。

## 体验指标建议
- 录入时长：单次录入 < 30 秒。
- 识别命中率：AI 关键信息填写准确率 > 80%。
- 跳出率：添加流程中断率下降。
