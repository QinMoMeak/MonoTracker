# MonoTracker 用户使用说明书

## 1. 文档说明

- 软件名称：MonoTracker
- 适用版本：1.5.15
- 适用对象：希望管理已购物品、心愿单、价格记录和消费趋势的个人用户
- 推荐平台：Android App
- 兼容平台：现代浏览器 Web 版、Capacitor 封装的 Android 版

本说明书基于当前项目源码、开发过程记录、`doc` 目录文档以及 DeepWiki 项目介绍页整理，内容以当前实际实现为准，重点面向软件使用者说明“能做什么、怎么做、遇到问题怎么办”。

---

## 2. 软件定位

MonoTracker 是一款本地优先的物品记录与资产追踪工具，适合以下场景：

- 记录自己已经购买的物品，管理价格、数量、来源、状态和使用情况
- 建立心愿单，持续观察目标商品的价格变化
- 借助 AI 识别图片和文字，减少手动录入成本
- 用统计图查看月度支出、持有时长、分类占比、渠道分布等信息
- 通过 ZIP 导入导出和 WebDAV 备份，保证数据可迁移、可恢复

软件强调两点：

1. 数据首先保存在本地  
2. 云端备份是可选能力，而不是强制依赖

## 3. 核心功能总览

### 3.1 物品管理
- 区分“已购买”和“心愿单”两类记录
- 支持新增、编辑、删除物品
- 支持图片、价格、数量、原价、分类、状态、渠道、店铺、备注、链接等字段
- 已购物品支持记录使用次数，并显示日均价值 / 次均价值

### 3.2 AI 快速录入
- 支持文字描述识别
- 支持上传单张或多张图片联合识别
- 识别后自动回填名称、价格、日期、分类、状态、渠道、店铺等字段
- 支持多家 AI 服务商切换

### 3.3 搜索与筛选
- 关键词搜索
- 日期区间筛选
- 价格区间筛选
- 已购买页可按状态筛选
- 心愿单页可按分类筛选

### 3.4 统计分析
- 总价值
- 物品数量
- 月度支出趋势
- 持有时长分布
- 状态分布
- 分类价值分布 / 数量分布
- 渠道支出分布

### 3.5 数据管理
- 自定义分类、状态、渠道
- CSV / ZIP 导入
- ZIP 导出
- 系统分享
- WebDAV 自动备份 / 手动上传 / 恢复

### 3.6 个性化设置
- 多语言：简体中文、繁体中文、English、日本語
- 主题色切换
- 外观模式切换：浅色 / 深色 / 跟随系统

## 4. 使用前准备

## 4.1 推荐使用方式

- 如果你需要 WebDAV 备份、系统分享、较稳定的本地文件存储，优先使用 Android App
- 如果你只是临时体验核心录入与统计功能，可以使用 Web 版

## 4.2 平台差异

### Android 版
- 数据保存在应用本地文件中
- 图片与主数据分开存储，启动更快
- 支持 WebDAV 备份
- 支持系统分享与原生文件保存

### Web 版
- 数据主要保存在浏览器本地存储
- 换浏览器、清理缓存、无痕模式可能导致数据不可保留
- WebDAV 因浏览器 CORS 限制不可用

## 4.3 网络要求

- 普通手动录入、查看统计、浏览本地数据：可离线使用
- AI 识别：需要联网
- WebDAV 备份与恢复：需要联网

## 5. 首次打开后的基本认识

底部导航共有 4 个标签页：

- 已购买：查看已拥有物品
- 心愿单：查看计划购买物品
- 统计：查看消费与资产数据
- 我的：设置、导入导出、AI、WebDAV、外观与语言

在“已购买”和“心愿单”页右下角有悬浮操作按钮：

- 主按钮：AI 已启用时为 AI 快速添加；AI 未启用时为普通新增
- 次按钮：AI 已启用时显示手动新增

## 6. 快速开始

## 6.1 手动添加一个已购物品

操作路径：

`已购买` -> 右下角新增按钮 -> 手动添加

建议至少填写以下字段：

- 名称
- 价格
- 购买日期
- 分类
- 状态

可选字段包括：

- 原价
- 数量
- 均价（会自动计算）
- 渠道
- 店铺
- 商品链接
- 备注
- 图片
- 价值显示方式（日均 / 次均 / 同时显示）

保存后，物品会按购买日期归入对应月份时间线中。

## 6.2 手动添加一个心愿单物品

操作路径：

`心愿单` -> 右下角新增按钮 -> 手动添加

与已购物品相比，心愿单更适合记录：

- 当前参考价格
- 目标商品图片
- 渠道 / 店铺 / 链接
- 历史价格记录

你可以在编辑表单中持续追加价格点，用于后续观察价格波动。

## 6.3 使用 AI 快速添加

操作路径：

`已购买` 或 `心愿单` -> 右下角 AI 按钮

支持两种输入：

- 文字描述
- 图片上传（支持多张）

推荐做法：

- 文字里尽量写清品牌、型号、价格、购买平台、日期
- 多张图片可以放商品图、订单页、价格页、规格页
- AI 识别完成后一定要进入手动表单复核，再保存

注意：

- 如果当前选择的模型不支持图片理解，图片识别可能失败
- AI 只负责辅助填表，不应视为绝对准确

## 7. 已购买页怎么用

已购买页主要用于管理已经拥有的物品。

### 7.1 你会看到什么
- 按月份分组的时间线
- 物品图片缩略图
- 价格、数量、均价
- 购买日期
- 状态、分类、渠道等标签信息
- 日均价值 / 次均价值

### 7.2 使用次数

已购物品支持“+1 次”记录使用次数。  
当你不断累计使用次数时，软件会重新计算次均价值，帮助你判断“这个物品到底值不值”。

### 7.3 编辑和删除

点击物品卡片可进入编辑。  
编辑页支持：

- 修改字段
- 更换图片
- 调整数量与均价
- 修改价值显示方式
- 删除当前物品

## 8. 心愿单页怎么用

心愿单适合做“待买物品池”和“价格观察列表”。

### 8.1 适合记录的内容
- 想买但还没买的商品
- 不同平台的报价
- 商品链接
- 预期购买日期
- 价格历史

### 8.2 历史价格记录

在心愿单编辑页中可以添加多条“日期 + 价格”记录。  
系统会保存这些价格点，并在列表或统计信息中用于展示：

- 最新价格
- 历史最低价
- 历史最高价
- 记录次数

适合用来比较：

- 是否到了值得入手的价格
- 某平台是否经常促销
- 某商品价格是否长期上涨或回落

## 10. 统计页怎么看

统计页不是记账报表，而是“物品资产视角”的分析页面。

## 10.1 资产概览
- 总价值：当前记录物品的总金额
- 物品数量：当前记录总条数

## 10.2 按月支出趋势

显示你在不同月份的支出变化。  
适合观察：

- 哪个月花得最多
- 是否存在集中消费周期
- 某些时间段是否冲动消费更明显

## 10.3 持有时长分布

把物品按持有时间归类，例如：

- 少于 1 个月
- 1–6 个月
- 6–12 个月
- 1–3 年
- 3 年以上

适合观察自己是否经常“买了很快闲置”。

## 10.4 其他分布统计

统计页还会展示：

- 状态分布
- 分类价值分布
- 分类数量分布
- 渠道支出分布

这些图表适合帮助你回答：

- 我最花钱的品类是什么
- 我最常买的东西是什么
- 我在哪个平台花的钱最多

## 12. AI 设置详细说明

操作路径：

`我的` -> `AI 设置`

## 12.1 当前支持的服务商

当前界面中可见的服务商包括：

- 禁用 AI
- OpenAI
- Google Gemini
- Anthropic
- DeepSeek
- Qwen
- Z.ai
- Doubao
- iFlow

服务商较多时，列表区域可以上下滚动查看全部选项。

## 12.2 AI 设置包含哪些内容

每个服务商都可以单独设置：

- API Key
- 模型
- Base URL

软件会记住你为不同服务商选择过的模型。

## 12.3 推荐的 iFlow 配置方式

如果你想使用 iFlow：

- 服务商：选择 `iFlow`
- API Key：填写你在 iFlow 控制台生成的密钥
- Base URL：可直接使用默认值 `https://apis.iflow.cn/v1`
- 模型：若主要用于图片识别，优先选择 `Qwen3-VL-Plus`

## 12.4 使用 AI 的建议

- 纯文本录入：可优先选响应稳定的文本模型
- 图片识别：优先选支持视觉能力的模型
- 多图分析：尽量上传清晰图片，避免模糊截图或重复截图

## 13. 数据管理

操作路径：

`我的` -> `数据管理`

这里可以维护三类基础数据：

- 分类
- 状态
- 渠道

支持操作：

- 新增
- 重命名
- 删除

删除时的处理逻辑：

- 删除分类：关联物品会回到“其他”
- 删除状态：关联物品会回到“全新”
- 删除渠道：关联物品的渠道会被清空

建议：

- 在正式大量录入前，先把自己常用的分类、状态、渠道整理好
- 这样后续统计结果会更稳定、更准确

## 14. 导入与导出

操作路径：

`我的` -> `导出数据` 或 `导入数据`

## 14.1 导出

导出面板支持三种方式：

- 保存为文件
- 系统分享
- 复制内容

实际导出内容说明：

- 保存文件 / 分享：导出为 ZIP 备份文件
- 复制内容：复制 CSV 文本，适合快速人工备份或粘贴到表格工具

ZIP 备份通常包含：

- `data.json`
- `data.csv`
- `images/`（若存在图片）

## 14.2 导入

支持导入：

- `.csv`
- `.zip`

导入时默认会尝试“合并”，不会粗暴清空现有数据。  
系统会优先按以下方式识别重复项：

1. 先按物品 ID
2. 再按关键字段签名（类型、名称、价格、日期、渠道、分类、状态）

如果识别为同一条记录，会更新原记录；否则会新增。

## 15. WebDAV 备份与恢复

操作路径：

`我的` -> `WebDAV 网盘`

这是软件的云端备份功能，推荐 Android App 使用。

## 15.1 WebDAV 基本设置

需要填写：

- 服务器地址
- 账户
- 密码

坚果云常见地址示例：

- `https://dav.jianguoyun.com/dav/`

应用会在远端使用固定根目录：

- `TrackerBackups/`

## 15.2 自动备份

可在 WebDAV 页面打开“自动备份”。

当前行为是：

- 每天第一次打开应用时自动尝试备份一次
- 只有成功提交后，才会记录当天已备份
- 如果当天已经成功备份，则不会重复自动上传

## 15.3 是否包含图片

WebDAV 页面有“包含图片”开关，默认关闭。

建议理解：

- 开启：备份更完整，但 ZIP 更大、上传更慢
- 关闭：备份更轻，适合日常频繁自动备份

如果你的图片很多，建议默认关闭；重要节点再手动做一次包含图片的完整备份。

## 15.4 恢复方式

恢复时可选两种模式：

### 合并到本地
- 适合不想丢失本地新增内容
- 会尽量把备份内容与本地记录合并

### 覆盖本地
- 会用备份内容直接替换当前本地数据
- 更适合换设备、回滚数据、或本地数据已经错乱时使用

使用覆盖恢复前，建议先导出一份当前本地备份。

## 15.5 备份历史显示规则

当前规则为：

- 恢复列表最多显示最近 3 次可用备份
- 云端最多保留最近 4 次备份
- 超出 4 次的旧备份会自动清理
- 未完成的备份不会显示为可恢复条目

## 15.6 Web 版限制

由于浏览器 CORS 限制，WebDAV 功能在浏览器中不可用。  
需要使用 Android App 才能正常上传、读取、恢复 WebDAV 备份。

## 17. 数据与隐私说明

## 17.1 本地优先

MonoTracker 默认把数据保存在本地：

- Web：浏览器本地存储
- Android：应用私有文件目录

在 Android 版中，图片已经与主状态分离存储，有利于：

- 提升启动速度
- 减少大量图片导致的卡顿
- 降低大体积写入引发的稳定性问题

## 17.2 AI 数据发送说明

只有在你主动使用 AI 快速添加时，输入的文本和图片才会发送给你选择的 AI 服务商。  
如果你不使用 AI，软件不会自动把本地物品数据上传给 AI 服务商。

## 17.3 WebDAV 数据说明

如果你启用了 WebDAV：

- 备份文件会上传到你配置的 WebDAV 空间
- 账户、密码和设置会保存在本地，用于后续自动备份或手动恢复

## 18. 常见问题

## 18.1 为什么我在 AI 设置里没看到 iFlow？

请确认：

- 当前应用版本是否为较新版本
- AI 服务商列表是否已经向下滚动查看到底

当前版本中，服务商区域支持独立滚动，`iFlow` 位于服务商列表中。

## 18.2 为什么 AI 识别失败？

常见原因包括：

- API Key 未填写
- Base URL 不正确
- 选择的模型不支持图片理解
- 当前网络异常
- 服务商接口限流或返回错误

建议先用纯文字测试，再切换到图片识别。

## 18.3 为什么 WebDAV 在浏览器里不能用？

因为浏览器 CORS 限制，当前实现只支持 Android 原生环境中的 WebDAV 读写。

## 18.4 导入后为什么不是完全覆盖？

普通导入是合并逻辑。  
如果你希望用云端备份替换本地内容，请在 WebDAV 恢复时选择“覆盖本地”。

## 18.5 为什么数据多了以后以前会卡？

旧实现中，图片和主状态写在同一份大数据里，容易在大量物品下变慢。  
当前版本已经把图片和主状态拆分存储，并优化了列表分批渲染与统计按需计算。

## 19. 使用建议

如果你想长期稳定使用，建议按下面方式操作：

1. 先整理好分类、状态、渠道  
2. 新增时优先保证名称、价格、日期准确  
3. 心愿单持续记录价格历史  
4. 每周或每月手动导出一份 ZIP  
5. Android 端开启 WebDAV 自动备份  
6. 日常自动备份可关闭“包含图片”，重要节点再做完整备份

---

## 20. 适合谁使用

MonoTracker 特别适合以下人群：

- 喜欢记录消费和物品的人
- 想减少冲动购物的人
- 需要管理大量数码、服饰、家居、美妆等个人资产的人
- 想把“物品管理”和“价格观察”结合起来的人
- 希望数据掌握在自己手里，而不是完全依赖云服务的人

---

## 21. 结语

MonoTracker 不是传统记账软件，也不是电商收藏夹。  
它更适合把“买过什么、值不值、用了多久、还想买什么、什么时候最划算”放在一个统一的本地工具里管理。

如果你愿意持续维护分类、状态、渠道与价格历史，这个软件的价值会随着使用时间逐渐放大。
