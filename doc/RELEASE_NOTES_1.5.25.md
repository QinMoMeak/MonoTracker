# Tracker 1.5.25

## 1.5.24 之后的修改汇总

### 图片新增稳定性修复
- 优化 Android 端添加带图片物品时的图片处理链路，减少大图解码、Canvas 导出和 Base64 传递带来的瞬时内存峰值
- 上传图片改为单图处理，并增加 8MB 限制，降低低内存设备在选图和保存阶段的崩溃概率
- 图片压缩参数下调为更保守的移动端配置，缩略图与保存图分离处理，同时在处理完成后主动释放 Canvas 与图片引用
- 编辑已有物品时不再把原始大图重新灌回弹窗状态，减少编辑与保存时的额外内存占用

### Android 返回与保存链路稳定性
- 保存物品后不再依赖浏览器历史回退关闭弹窗，避免 Android WebView 中重复退栈引发的异常退出
- 弹窗与页面覆盖层改为统一状态关闭路径，收敛返回键处理逻辑，降低连续操作时的状态竞争风险
- Android 原生端保存流程停止镜像写入 localStorage，改为以 Filesystem 为主，减少一次大状态序列化和图片重复写入

### 列表切换与图片加载优化
- 高频切换页面时，非激活标签页停止图片懒加载与额外观察器工作，减少后台无效任务
- 列表预取图片增加延迟与取消逻辑，并加入读图并发上限，降低快速切页时的资源堆积
- 新增物品与导入数据的 ID 生成改为优先使用 UUID，减少极快连续操作下的碰撞风险

### 日均价值功能增强
- 为物品新增“最后使用日期”字段，可用于更准确地计算日均价值
- 当价值显示模式为“仅日均价值”时，新增/编辑弹窗会显示“最后使用日期”输入项，未填写时仍可正常保存
- 日均价值计算改为优先基于“最后使用日期”推导已使用天数；如果未填写，则自动回退到购买日期
- 时间线卡片在“仅日均价值”模式下新增“已使用天数”展示，与左侧日均价值信息并排显示

### 构建与验证
- Android 版本更新为 1.5.25（versionCode 44）
- 已完成前端生产构建、Capacitor Android 同步与 Debug APK 构建验证

## 说明
- 本次版本以“带图新增稳定性修复 + 日均价值体验补强”为主，重点面向 Android 端实际使用场景
